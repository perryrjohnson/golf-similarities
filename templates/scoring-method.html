Scoring Methodology

Learn More about this project in this Write-Up

We're focused on finding similar PGA Tour players solely on their shotmaking results and performance across statistical categories. This means that instead of categorizing players by number of wins, world rankings, and earnings, we are looking at the statistical inputs: Driving Distance, Driving Accuracy, Greens in Regulation, Sand Save Percentage, and Strokes Gained Putting, etc.

  • To build the machine learning model for calculating similarity among players, I leveraged Apple's open-source machine learning library, Turi Create. The Item Content Similarity machine learning model only takes into account the player statistics. It looks at each distinct pair of players and calculates how similar they are based on the statistical attributes. This similarity score is calculated by first computing the similarity between each feature and then takes a weighted average of those to get the final similarity. This is useful because one can specify a player and this model will find the players with the closest statistical performance. Example: "If you select Jordan Spieth, here are PGA Tour players that have the most similar statistics to Jordan Spieth this season."
